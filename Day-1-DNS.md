Understanding DNS (Domain Name System)
What I Learned Today

Today I studied the basics of DNS (Domain Name System), which is often called the "phonebook of the internet."

When we type a website name such as google.com, computers cannot directly understand domain names. They communicate using IP addresses. DNS translates human-readable domain names into IP addresses so browsers can connect to the correct server.

How DNS Works
User enters a domain name in the browser.
The browser checks its local cache.
If not found, the request goes to a DNS Resolver.
The resolver queries DNS servers to find the correct IP address.
The IP address is returned to the browser.
The browser connects to the website's server.
Common DNS Records
A Record

Maps a domain name to an IPv4 address.

Example:
example.com → 192.168.1.1

AAAA Record

Maps a domain name to an IPv6 address.

CNAME Record

Creates an alias for another domain.

MX Record

Specifies mail servers responsible for receiving emails.

TXT Record

Stores text information, often used for verification and security purposes.

Why DNS is Important in Cybersecurity
DNS helps systems locate services on the internet.
Attackers may abuse DNS through:
DNS Spoofing
DNS Cache Poisoning
DNS Tunneling
DNS Amplification Attacks

Understanding DNS is essential for network security and cybersecurity professionals.

Key Takeaway

DNS is a critical internet service that translates domain names into IP addresses. A strong understanding of DNS helps cybersecurity professionals detect, analyze, and prevent various network-based attacks.

# DNS (Domain Name System) Fundamentals

## What I Learned Today

Today I learned about DNS (Domain Name System), a core component of the internet that translates human-readable domain names into IP addresses.

For example, when a user types `google.com` into a browser, DNS helps find the corresponding IP address so the browser can connect to the correct server.

## DNS Resolution Process

1. User enters a domain name.
2. Browser checks its cache.
3. Request is sent to a DNS Resolver.
4. Resolver queries DNS servers.
5. Correct IP address is returned.
6. Browser connects to the target server.

## Common DNS Records

- **A Record** → Maps a domain to an IPv4 address.
- **AAAA Record** → Maps a domain to an IPv6 address.
- **CNAME Record** → Creates an alias for another domain.
- **MX Record** → Handles email routing.
- **TXT Record** → Stores verification and security-related information.

## DNS and Cybersecurity

DNS plays a critical role in cybersecurity. Some common DNS-related attacks include:

- DNS Spoofing
- DNS Cache Poisoning
- DNS Tunneling
- DNS Amplification Attacks

Understanding DNS is essential for identifying and mitigating network-based threats.

## Key Takeaway

DNS acts as the internet's directory service, converting domain names into IP addresses. A strong understanding of DNS is fundamental for anyone pursuing cybersecurity and networking.
